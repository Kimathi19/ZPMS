# Zin Pharmacy Management System (ZPMS)

## Overview

The **Zin Pharmacy Management System (ZPMS)** is a web-based pharmacy management solution developed to streamline day-to-day pharmacy operations. The system was designed and implemented as an academic degree thesis and practical project, focusing on improving efficiency, accuracy and accountability in pharmacy workflows.

ZPMS centralizes core pharmacy processes such as product management, sales transactions, and record keeping, reducing reliance on manual methods and minimizing operational errors.

---

## Project Objectives

The main objectives of ZPMS are to:

* Automate pharmacy sales and transaction processing
* Improve accuracy in stock handling and record keeping
* Provide clear transaction history and reporting
* Enhance operational efficiency for pharmacists and attendants
* Offer a simple, intuitive user interface for daily use

---

## Key Features

* **User Authentication** – Secure login for authorized users
* **Product Management** – Add, update, and manage pharmacy products
* **Transaction Processing** – Cart-based sales with automatic stock reduction
* **Sales Recording** – All completed transactions are stored for reference
* **Receipt Generation** – Support for generating sample/print receipts
* **Dashboard Overview** – Quick access to essential pharmacy data
* **Responsive Frontend** – Optimized for desktop and tablet use

---

## System Architecture

ZPMS follows a modern web-based architecture:

* **Frontend:** Web interface for user interaction
* **Backend / Services:** Handles business logic and data operations
* **Database:** Cloud-based storage for products, users, and transactions

The system was designed with scalability and maintainability in mind.

---

## Technology Stack

* **Frontend:** HTML, CSS, JavaScript (framework-based)
* **Backend:** Firebase services
* **Database:** Firestore (NoSQL)
* **Authentication:** Firebase Authentication
* **Hosting:** Vercel

## How to Run Locally

1. Clone the repository

   ```bash
   git clone <repository-url>
   ```
2. Navigate to the frontend directory

   ```bash
   cd frontend
   ```
3. Install dependencies

   ```bash
   npm install
   ```
4. Start the development server

   ```bash
   npm run dev
   ```

* No integration with supplier systems
* Limited role-based access control
